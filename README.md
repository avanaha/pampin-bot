# PamPin Bot 🤖

**PamPin** — твой календарь важных дат для мессенджера МАКС. Напомнит обо всём, что ты расскажешь.

## 📋 Возможности

- ✅ Создание напоминаний о важных датах
- ✅ Гибкие периоды напоминаний (за 3 месяца, за день, за час и т.д.)
- ✅ Ежегодное повторение напоминаний
- ✅ Поддержка различных часовых поясов России
- ✅ Уведомления в закрытую группу
- ✅ Интуитивный интерфейс с inline-клавиатурой

## 🚀 Установка и запуск

### Локальная разработка

1. **Клонируйте репозиторий:**
```bash
git clone https://github.com/yourusername/pampin-bot.git
cd pampin-bot
```

2. **Установите зависимости:**
```bash
npm install
```

3. **Создайте файл `.env`:**
```env
MAX_BOT_TOKEN=ваш_токен_бота
GROUP_ID=-72385180944913
TIMEZONE=Europe/Moscow
DATABASE_PATH=./data/pampin.db
NODE_ENV=development
```

4. **Запустите в режиме разработки:**
```bash
npm run dev
```

### Docker

```bash
# Сборка
docker build -t pampin-bot .

# Запуск
docker run -d \
  --name pampin-bot \
  -e MAX_BOT_TOKEN=ваш_токен \
  -e GROUP_ID=-72385180944913 \
  -v $(pwd)/data:/app/data \
  pampin-bot
```

### Docker Compose

```bash
docker-compose up -d
```

## 🌐 Деплой на Amver.ru

1. **Создайте проект на Amver.ru**

2. **Подключите GitHub репозиторий**

3. **Настройте переменные окружения:**
   - `MAX_BOT_TOKEN` — токен бота
   - `GROUP_ID` — ID группы для уведомлений
   - `WEBHOOK_URL` — URL для вебхуков (опционально)
   - `NODE_ENV=production`

4. **Amver автоматически определит Dockerfile и запустит бота**

## 📖 Использование

### Команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Показать главное меню |
| `/list` | Список ваших напоминаний |
| `/settings` | Настройки (часовой пояс) |
| `/help` | Справка |

### Создание напоминания

1. Нажмите **"➕ Добавить напоминание"**
2. Введите название события
3. Укажите дату (форматы: `25.12.2025`, `25 декабря`, `завтра`)
4. Укажите время (например: `14:30`)
5. Добавьте описание (опционально)
6. Выберите периоды напоминаний
7. Подтвердите создание

### Периоды напоминаний

Можно выбрать несколько периодов:
- за 1 год
- за 6 месяцев
- за 3 месяца
- за 1 месяц
- за 2 недели
- за 1 неделю
- за 3 дня
- за 1 день
- за 12 часов
- за 6 часов
- за 3 часа
- за 1 час
- за 30 минут

### Поддерживаемые форматы даты

- `25.12.2025` — день.месяц.год
- `25/12/2025` — день/месяц/год
- `2025-12-25` — год-месяц-день
- `25 декабря 2025` — текстовый формат
- `завтра` — относительная дата
- `через неделю` — относительная дата
- `в следующем месяце` — относительная дата

### Часовые пояса

Поддерживаются все часовые пояса России:
- Москва (UTC+3)
- Калининград (UTC+2)
- Самара (UTC+4)
- Екатеринбург (UTC+5)
- Омск (UTC+6)
- Красноярск (UTC+7)
- Иркутск (UTC+8)
- Якутск (UTC+9)
- Владивосток (UTC+10)
- Магадан (UTC+11)
- Камчатка (UTC+12)

## 🔧 Переменные окружения

| Переменная | Обязательно | Описание |
|------------|-------------|----------|
| `MAX_BOT_TOKEN` | ✅ | Токен бота MAX |
| `GROUP_ID` | ✅ | ID группы для уведомлений |
| `WEBHOOK_URL` | ❌ | URL для вебхуков (production) |
| `PORT` | ❌ | Порт сервера (по умолчанию 3000) |
| `TIMEZONE` | ❌ | Часовой пояс (по умолчанию Europe/Moscow) |
| `DATABASE_PATH` | ❌ | Путь к базе данных |
| `NODE_ENV` | ❌ | Окружение (development/production) |

## 🛠 Технологии

- **Node.js 20** — среда выполнения
- **TypeScript** — типизированный JavaScript
- **better-sqlite3** — SQLite база данных
- **chrono-node** — парсинг дат на естественном языке
- **node-cron** — планировщик задач

## 🔐 Безопасность

- Токен бота передается через заголовок Authorization
- База данных SQLite хранится локально
- Все данные шифруются при передаче по HTTPS
